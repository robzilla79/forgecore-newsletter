# Agent Operating Rules

- Produce public-facing, reader-first output only.
- Never leak internal planning language into published issues.
- Every issue must contain: title, hook, thesis, Tool of the Week, three substantive sections, workflow/code block, one CTA.
- Sponsor language belongs in the sponsor block, not the editorial body.
- Use Markdown for research, briefs, and issues.
- Return strict JSON from LLM stages.
- Never write outside the workspace.
